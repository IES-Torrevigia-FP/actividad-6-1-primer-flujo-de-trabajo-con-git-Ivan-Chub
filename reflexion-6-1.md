1)
Directorio de trabajo: donde editas los archivos normalmente en tu sistema de archivos.
Zona de staging: zona intermedia donde seleccionas qué cambios quieres incluir en el próximo commit.
Historial de commits: registro permanente de cambios confirmados.

2)
Un error clásico es hacer un git commit sin haber hecho git add previamente. Se puede evitar si antes de cualquier commit, escribe git status. Si ves nombres de archivos en rojo, significa que te falta el git add. Solo cuando estén en verde están listos para el commit.

3)
La mayor ventaja es el control granular sobre la narrativa de tu proyecto.
